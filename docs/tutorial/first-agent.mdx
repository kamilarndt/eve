---
title: "1. Create the Agent"
description: "Scaffold a local analytics agent, configure a model credential, and verify its first response."
---

This tutorial builds a small analytics agent from a clean directory. By the end, it will inspect
sample orders, delegate a review, create a report in its sandbox, remember a metric definition,
load a playbook, request approval, and run in a local browser UI.

We keep the required path deliberately local and dependency-light. You do not need a database,
private beta, deployment, Python, or a non-default sandbox binary.

## Prerequisites

- Node.js 24 or newer.
- npm.
- Either `AI_GATEWAY_API_KEY`, a Vercel OIDC credential, or a direct-provider key supported by the model you select.

## Scaffold

Run from the directory that should contain the new project:

```bash
npx eve@latest init analytics-assistant
cd analytics-assistant
```

The initializer selects a model, writes the project, installs dependencies, and may start
`eve dev`. Stop that process before editing files in later steps; we will restart it after each
meaningful capability so you can verify the project incrementally.

If you use AI Gateway outside a linked Vercel environment, create `.env.local` in `analytics-assistant/`:

```dotenv title=".env.local"
AI_GATEWAY_API_KEY=your_gateway_key
```

Do not commit this file.

## Add standing instructions

Replace `agent/instructions.md`:

```md title="agent/instructions.md"
You are a data analyst working from the data and tools available to you.

- Compute answers instead of guessing.
- State the date range and units behind every number.
- Distinguish facts from assumptions.
- If the available data cannot answer a question, say what is missing.
```

## Run and verify

From `analytics-assistant/`:

```bash
npm run dev
```

The default server listens on port `2000`; if that port is occupied, use the URL printed by `eve dev`. In the terminal UI, send:

```text
What information do you need to calculate revenue by customer?
```

The reply should ask for order or transaction data and should not invent totals. A provider authentication error means the model credential is missing or does not match the model in `agent/agent.ts`; fix that before continuing.
